# Tool: Google Veo 3 (Vertex AI direct)

## What it does
Google's flagship AI video generation model, available natively on Vertex AI: text-to-video and image-to-video at 720p/1080p, 4/6/8s clips, 9:16 or 16:9, with native synchronized audio (sound effects, dialogue, ambient) in a single inference pass.

---

## When to choose Vertex direct vs Replicate pass-through

**The cost is identical.** As of 2026-05-06, Vertex AI direct and Replicate charge the same published rates for Veo 3:

| Model | Video + Audio | Video only |
|---|---|---|
| Veo 3.0 (`veo-3.0-generate-001`) | ~$0.75/s | ~$0.50/s |
| Veo 3.0 Fast (`veo-3.0-fast-generate-001`) | ~$0.40/s | ~$0.30/s |
| Veo 3.1 (`veo-3.1-generate-001`) | ~$0.40/s | ~$0.30/s |
| Veo 3.1 Fast (`veo-3.1-fast-generate-001`) | ~$0.15/s | ~$0.10/s |
| Veo 3.1 Lite (`veo-3.1-lite-generate-001`) | ~$0.05/s | ~$0.05/s |

Replicate's blog comparison (published August 2025) shows $0.75/s for Veo 3 Standard and $0.40/s for Veo 3 Fast — matching Vertex direct. There is no arbitrage between the two surfaces.

**Choose Vertex direct when:**
- You already have a GCP service account and want billing under one Google Cloud invoice.
- You need provisioned throughput (guaranteed capacity via GSU purchase) — Replicate has no equivalent.
- You want SynthID digital watermarks and Content Credentials (C2PA) tracked through Google Cloud.
- You need image-to-video (`generateVideos` with an image input URI).
- You are already using `@google/genai` SDK for Gemini — the same client handles Veo.

**Choose Replicate pass-through when:**
- You have existing Replicate credits/billing and don't want a GCP account.
- You need a simpler REST call with no GCP IAM setup (Replicate auth = single bearer token).
- You want to compare Veo against Kling, Hailuo, Seedance in the same API surface.

**Feature parity note:** Replicate hosts `veo-3-fast` and selected Veo variants, but not all model versions (e.g., `veo-3.1-lite-generate-001` is Vertex-only as of 2026-05-06). For the full model menu — especially newer Veo 3.1 variants — Vertex direct wins.

**Latency:** Replicate adds a thin proxy layer (~0–5 seconds overhead). For 8-second clips with ~90s generation time (Veo 3 Standard), this is negligible.

**Audio sync availability:** Native synchronized audio ships with Veo 3 and all Veo 3.1 variants on Vertex. The `generateAudio` flag is **not exposed as a controllable parameter via the Gemini API surface** (January 2026 forum report: "generate_audio parameter is not supported in Gemini API"). On Vertex via `@google/genai`, audio is generated by default. To suppress audio, use negative prompting ("no background music, no dialogue, ambient silence only") or switch to Veo 3.1 Fast, which is more likely to produce silent clips per community reports.

## Auth + endpoint (verified 2026-05-06)

**Vertex AI predictionService endpoint:**
```
us-central1-aiplatform.googleapis.com
```

**Model IDs (use these exact strings):**

| Model | ID | Status |
|---|---|---|
| Veo 3.0 Standard | `veo-3.0-generate-001` | GA (launched 2025-07-29, discontinues 2026-06-30) |
| Veo 3.0 Fast | `veo-3.0-fast-generate-001` | GA (launched 2025-07-29, discontinues 2026-06-30) |
| Veo 3.1 Standard | `veo-3.1-generate-001` | GA |
| Veo 3.1 Fast | `veo-3.1-fast-generate-001` | GA |
| Veo 3.1 Lite | `veo-3.1-lite-generate-001` | Preview (text-to-video only) |

Preview model IDs (`veo-3.0-generate-preview`, `veo-3.0-fast-generate-preview`) were deprecated 2026-04-02 — do not use them.

**Auth pattern:** Application Default Credentials (ADC) — the SDK picks up the service account JSON automatically when `GOOGLE_APPLICATION_CREDENTIALS` points to it. No manual token exchange required.

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/viewer@ryanrealty.iam.gserviceaccount.com.json"
export GOOGLE_CLOUD_PROJECT="ryan-realty-tc"
export GOOGLE_CLOUD_LOCATION="us-central1"
```

**Required IAM role:** `roles/aiplatform.user` — already granted to `viewer@ryanrealty.iam.gserviceaccount.com` per task brief.

**GCS bucket required:** Veo returns output as a GCS URI, not base64. You must supply an output bucket:
```
outputGcsUri: "gs://ryan-realty-video-output/veo3/"
```
The service account needs `roles/storage.objectCreator` on that bucket.

**SDK package:**
```bash
npm install @google/genai
```
Note: `@google/genai` (not `@google-cloud/aiplatform`) is the recommended SDK for Veo as of mid-2025. The older `@google-cloud/vertexai` package also works but the canonical examples now use `@google/genai`.

## Node.js SDK pattern

```javascript
import { GoogleGenAI } from '@google/genai';

const client = new GoogleGenAI({
  vertexai: true,
  project: process.env.GOOGLE_CLOUD_PROJECT,     // 'ryan-realty-tc'
  location: process.env.GOOGLE_CLOUD_LOCATION,   // 'us-central1'
});

async function generateRealEstateVideo(prompt, outputGcsUri, imageGcsUri = null) {
  const config = {
    aspectRatio: '9:16',          // portrait for social
    outputGcsUri,
    sampleCount: 1,               // 1–4 videos per call
    durationSeconds: 8,           // 4, 6, or 8
    // No generateAudio flag — audio is on by default; suppress via prompt if needed
  };

  // Image-to-video: attach source photo
  if (imageGcsUri) {
    config.image = {
      gcsUri: imageGcsUri,
      mimeType: 'image/jpeg',
    };
  }

  let operation = await client.models.generateVideos({
    model: 'veo-3.1-fast-generate-001',  // use Fast for iteration; Standard for finals
    prompt,
    config,
  });

  // Poll until done (typically 60–120s)
  while (!operation.done) {
    await new Promise(r => setTimeout(r, 15000));
    operation = await client.operations.get({ operation });
  }

  if (!operation.response?.generatedVideos?.length) {
    throw new Error('Veo generation failed: ' + JSON.stringify(operation.error));
  }

  return operation.response.generatedVideos[0].video.uri; // GCS URI
}
```

**Response shape:**
```json
{
  "generatedVideos": [
    {
      "video": {
        "uri": "gs://ryan-realty-video-output/veo3/output_0.mp4"
      }
    }
  ]
}
```

After generation, download with `gcloud storage cp` or the GCS Node.js client and pass to the Remotion pipeline.

## Quotas + rate limits (verified 2026-05-06)

| Model tier | Requests per minute (regional) |
|---|---|
| GA models (3.0 and 3.1) | 50 requests/minute/model |
| Preview models | 10 requests/minute/model |

- Dynamic shared quota is **not supported** for any Veo model — you must operate under fixed quota or provisioned throughput.
- Region: `us-central1` is confirmed supported. "Global" endpoint also works in some SDK examples but `us-central1` is safest for `ryan-realty-tc`.
- `sampleCount` up to 4 per request (GA models).

At 50 RPM with 1–4 samples per call, sustained throughput ceiling is ~200 videos/minute — far above any realistic Ryan Realty volume.

## Prompt template

```
[SUBJECT] in [LOCATION DETAIL], [CAMERA MOVE], [LIGHTING], [ATMOSPHERE].
No text overlays. No watermarks. Photorealistic.
[AUDIO NOTE if needed]
```

Real estate-specific camera moves that Veo 3.1 honors reliably:
- `slow dolly forward` / `slow dolly backward`
- `gentle pan left to right`
- `crane shot rising slowly`
- `orbit around subject at constant distance`
- `push in on [specific element]`

## Common failure modes

**1. Audio always on — no kill switch via API flag**
The `generateAudio` / `generate_audio` boolean is documented in some places but throws an error when passed via the Gemini API surface (confirmed January 2026). Use negative prompt language instead. Workaround: render with audio, then mux in your own audio track during Remotion/ffmpeg post-processing (strip the Veo audio with `-an`, add ElevenLabs VO + music bed).

**2. GCS output bucket permissions**
If the service account lacks `roles/storage.objectCreator` on the output bucket, the operation will complete but silently fail to write the file. The `operation.done` flag returns `true` but `generatedVideos` is empty or missing. Always pre-verify bucket ACL before sending production requests.

**3. Quota exhaustion (50 RPM)**
At burst rates (batch listing video production), you can hit the 50 RPM wall. Implement exponential backoff starting at 5s. Do not retry faster than 15s — Veo operations are async and the server does not benefit from rapid polling.

**4. Region mismatch**
`us-central1` is the confirmed location for `ryan-realty-tc`. Specifying `global` works in some SDK versions but can route requests to a region without Veo capacity. Hardcode `us-central1`.

**5. Model discontinuation (veo-3.0 EOL: 2026-06-30)**
`veo-3.0-generate-001` and `veo-3.0-fast-generate-001` both discontinue 2026-06-30. Migrate to `veo-3.1-generate-001` / `veo-3.1-fast-generate-001` now. The Veo 3.1 variants are GA and price-equivalent to Veo 3.0 Fast.

**6. Prompt language — English only**
Veo 3 accepts English prompts only. If localizing for Spanish-language social content, write the prompt in English; the generated dialogue/audio can be in other languages based on scene context.

**7. No 4K on GA models**
4K resolution is listed as "preview" on Veo 3.1 only — not available on GA models as of 2026-05-06. GA resolution ceiling is 1080p. Do not promise 4K output to clients without checking current model capabilities.

## Output format

- **Container:** `video/mp4`
- **Delivery:** GCS URI (`gs://bucket/path/output.mp4`) — not base64, not a signed URL
- **Resolution:** 720p or 1080p (GA models); 4K (Veo 3.1 preview only)
- **Frame rate:** 24 FPS
- **Duration options:** 4, 6, or 8 seconds
- **Audio:** AAC stereo mixed into the MP4 container when audio is generated
- **Watermark:** SynthID embedded invisibly on every frame (not visible, not removable)
- **C2PA:** Content Credentials metadata attached

To use in Remotion pipeline:
```bash
# Pull from GCS
gcloud storage cp gs://ryan-realty-video-output/veo3/output.mp4 listing_video_v4/out/veo3_source.mp4

# Strip Veo audio (if using ElevenLabs VO instead)
ffmpeg -i listing_video_v4/out/veo3_source.mp4 -an listing_video_v4/out/veo3_silent.mp4
```
