# Tool: Synthesia (AI Avatar Video)

**Verified:** 2026-05-06 — sources: synthesia.io/pricing, docs.synthesia.io/reference, docs.synthesia.io/docs/personal-avatars, docs.synthesia.io/docs/synthesia-avatars, wavgroup.com AI labeling guide, Oregon SB 1546, California SB 942, FTC AI guidance.

---

## What it does

Converts a plain-text script into a 1080p MP4 where a photorealistic AI avatar speaks the words on camera. The avatar's lip-sync, facial expressions, and head movement are generated by Synthesia's cloud renderer — no camera, no studio, no ElevenLabs VO pass required. Output is a downloadable MP4. We post-process it via Remotion to add brand overlays, stat pills, end cards, and the mandatory AI disclosure pill.

| Use case | Synthesia avatar | Veo 3 / Kling i2v | ElevenLabs + photo b-roll | Human-shot video |

| Recurring "anchor" presenter (weekly market pulse) | **Best** | Not suitable | Acceptable fallback | Gold standard but expensive |
| Market report with talking-head authority | **Best** | No — no talking head | Acceptable | Expensive |
| Cinematic listing tour | Not suitable | **Best** | Not suitable | Preferred |
| News clip (no presenter) | Not applicable | **Best** | Good | Good |
| Agent intro / bio video | Personal avatar if licensed | Not suitable | Not suitable | **Best** |

**Use Synthesia when you need a consistent human presenter face, repeating weekly at scale, without booking talent. Do NOT use for cinematic property footage — AI avatars in listing tours read as cheap.**

Note: The existing `app/actions/synthesia.ts` passes the key directly (`Authorization: apiKey`) without the `Bearer ` prefix. The official docs and third-party guides show both patterns; the codebase uses the direct key pattern — do not change it unless Synthesia returns 403.

### Core endpoints

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/v2/videos` | Create video (async — returns ID immediately) |
| GET | `/v2/videos/{id}` | Poll status; response includes `download` URL when `status === "complete"` |
| GET | `/v2/videos` | List videos (filter by `source=my_videos\|workspace\|shared_with_me`) |
| GET | `/v2/templates` | List templates |
| POST | `/v2/dubbing` | Dub existing video into another language |
| GET/POST | `/v2/assets` | Upload/manage image, video, audio assets |

### Video creation payload

```json
{
  "title": "Bend Market Update — Week of 2026-05-05",
  "visibility": "private",
  "test": false,
  "aspectRatio": "9:16",
  "input": [
    {
      "avatar": "anna_costume1_cameraA",
      "scriptText": "Bend real estate. Week of May fifth...",
      "background": "office_branded"
    }
  ]
}
```

Key fields:
- `aspectRatio` — `"9:16"` for Reels/TikTok/Shorts portrait. Also supports `16:9`, `1:1`, `4:5`, `5:4`.
- `test: true` — watermarked render, does not count against minute quota. Use for QA.
- `input[].avatar` — avatar ID string (see Avatar Selection below).
- `input[].scriptText` — plain text. 180 words max for 45s delivery. Synthesia handles TTS internally.
- `input[].background` — background slug. `"white_studio"` is the safe fallback; `"office_branded"` if configured in account.
- `callbackId` — optional webhook correlation ID.

### Status response fields

```json
{
  "id": "abc123",
  "status": "complete",
  "download": "https://share.synthesia.io/videos/abc123.mp4"
}
```

Status values: `in_progress` → `complete` | `error` | `rejected` | `deleted`. The `download` field appears only when `status === "complete"`. Current polling implementation: 20 attempts × 15s = 5-minute max wait (sufficient for most renders; enterprise renders typically complete in 3-8 minutes for a 60s video).

Webhook alternative: set `callbackId` and configure a webhook endpoint in the Synthesia dashboard to receive `video.completed` and `video.failed` events — eliminates polling.

## Script structure for our use case

**Target: 45-second avatar window → 150–180 words.**

ElevenLabs is NOT used for this format. Synthesia handles TTS internally with its own voice models (or Matt's cloned voice if the personal avatar includes voice cloning). The pronunciation rules that apply to ElevenLabs (spelling "duh-shoots" for Deschutes) may differ for Synthesia's TTS — test each place name before production:

| Place name | ElevenLabs input | Synthesia TTS test required |
|------------|------------------|-----------------------------|
| Deschutes | "duh-shoots" | Test in watermarked render |
| Tumalo | "TOO-muh-low" | Test |
| Awbrey Butte | Standard | Test |
| Tetherow | Standard | Test |

**Script format (first-person, Matt's avatar):**

```
Bend real estate. Week of [month day].

The median close price is [price]. That is [X] percent [up/down] from one year ago.

Homes are averaging [N] days on market. [Faster/Slower] than last quarter.

[N] active listings. [N]-month supply. [Market classification: seller's, balanced, or buyer's market].

Sale-to-list ratio: [N] percent. Sellers are getting [N] percent of asking.

Bottom line: [one direct factual sentence derived from the data only].

For the full breakdown, visit ryan-realty.com.
```

**Script format (third-person, stock avatar):**

Replace "I" framing with "Ryan Realty's data shows..." and "the market shows...". Never use "I" with a stock avatar that is not Matt's likeness.

**Pause / emphasis tags:** Synthesia does not natively support SSML pause tags in the plain-text API input. For natural pacing, end sentences with a period and use short sentences (12 words max). Line breaks in `scriptText` are treated as brief pauses.

**Numbers — spell out for TTS:** Use "four hundred seventy-five thousand dollars" not "$475,000" in `scriptText`. Stat overlays in Remotion display the formatted "$475K" — the script is audio only.

**Background selection:**
- `"white_studio"` — clean, neutral, works on all plans
- `"office_branded"` — professional office setting (check availability in account)
- `"green_screen"` — allows Remotion to key in custom background post-render

For market pulse videos: `"white_studio"` or a solid branded background is preferred. Green screen + Remotion keying adds complexity; use only if brand guidelines require a specific custom background.

## Mandatory AI disclosure (verified 2026-05-06)

### Legal basis — three overlapping frameworks

**1. FTC (federal):**
- AI-generated content used in commercial advertising requires "clear and conspicuous" disclosure.
- For video: a persistent on-screen text overlay visible for at least 3–5 seconds at the start of the avatar segment, OR a lower-third that runs the full duration. Both meet the standard.
- Violations: up to $51,744 per violation.

**2. California SB 942 (effective January 1, 2026):**
- Requires embedded metadata (`AI_Generated: True`, provider name, timestamp, unique content ID) in AI-generated video.
- Requires a visible watermark or label (e.g., "Generated with AI").
- Applies to California distribution — which includes any content posted to Instagram, TikTok, or YouTube where California residents will see it.
- Penalty: up to $5,000 per day for removing or failing to maintain disclosure features.

**3. Oregon SB 1546 (signed, effective January 1, 2027):**
- Targets AI companions (chatbots with persistent relationships) — not directly applicable to one-way video content.
- Oregon's Unlawful Trade Practices Act (existing law) applies: representing AI-generated content as human-created content is an unlawful trade practice.
- NAR Code of Ethics and Oregon real estate license law require truthful advertising — an undisclosed AI avatar implying it is a real human presenter is a compliance risk.

**4. NAR / Oregon Real Estate Agency:**
- No avatar-specific rule as of 2026-05-06, but general advertising truthfulness requirements apply.
- Best practice: treat the avatar disclosure the same way you treat virtual staging disclosure — visible, early, unambiguous.

### Disclosure implementation (non-negotiable)

Synthesia does NOT add disclosure labels natively. Remotion adds the pill in post-processing.

**Pill spec (from `avatar_market_update/SKILL.md` §6, confirmed against CLAUDE.md brand rules):**

```
Text:       "AI avatar"
Font:       AzoSans, 20px, white
Background: rgba(16, 39, 66, 0.70) — navy at 70% opacity
Position:   bottom-left of avatar segment, inside safe zone (x=90, y=1700)
Corner radius: 12px
Duration:   First 5 seconds of avatar segment only (seconds 5–10 of full video)
```

**Why first 5 seconds only (not full duration):** FTC "clear and conspicuous" is satisfied by prominent early disclosure. A full-duration pill competes with stat overlays and distracts from the data delivery. First-5-second placement matches industry practice (same as "paid partnership" disclosure on Instagram Reels). California SB 942 is satisfied by the embedded metadata + the early visible label.

**Metadata embed (for California SB 942 compliance):**
Add the following to the final ffmpeg encode pass:

```bash
ffmpeg -i avatar_market_render.mp4 \
  -metadata comment="AI_Generated: True; Provider: Synthesia; Generator: Ryan Realty avatar_market_update pipeline" \
  -metadata date="$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
  -c copy \
  avatar_market_final.mp4
```

## Common failure modes

| Failure | Symptom | Fix |
|---------|---------|-----|
| Wrong avatar rendered | Synthesia substitutes a default avatar when the requested ID is invalid or unavailable on the plan | Always validate `SYNTHESIA_AVATAR_ID` is accessible on the account before submitting; use `test: true` first |
| Pronunciation drift on Oregon place names | "Deschutes" rendered as "Des-chutes" (hard ch) | Test each place name in a watermarked render before production; adjust spelling in `scriptText` until correct (e.g., "Duh-shoots" or "Des-shoots") |
| Mouth-shape artifacts at fast speech | Uncanny lip sync at > 180 words/60s | Keep scripts at 150–165 words for 45s; leave end punctuation, do not use comma-heavy run-ons |
| Render takes > 30 minutes | API job stalled | Log to `render_error.log`, alert — do not silently retry with a different avatar; check Synthesia status page at status.synthesia.io |
| Audio out of sync with overlays | Synthesia's word-level timestamps not captured | `render_synthesia.py` must capture timestamps from the API response; stat overlays key off those timestamps, not clock time |
| Test render not watermarked (charged to quota) | Accidental production render during QA | Always set `test: true` for the first render of any new script or avatar ID |
| Personal avatar sharing blocked | Error when another workspace member tries to use Matt's avatar | Personal avatars from photo cannot be shared; video-based personal avatars are Enterprise-only for sharing; Matt's login must submit the render |

## Output format

Synthesia returns a `download` URL to an MP4 file hosted on Synthesia's CDN. The file is private (not publicly accessible without the signed URL). Download immediately after `status === "complete"` — the CDN link may expire.

```
out/avatar_market_update/Bend_<yyyy-mm-dd>/
  synthesia_raw.mp4        # Downloaded from Synthesia CDN (avatar segment, ~40-50s)
  synthesia_qa_frames/     # ffmpeg frame extractions for QA
  avatar_market_render.mp4 # Remotion output (full 60s: 5s intro + 45s avatar + 5s CTA + 5s contact)
  avatar_market_final.mp4  # CRF 24 encode with SB 942 metadata embedded
  render_log.txt           # Avatar ID, video ID, API response metadata, timestamps
  render_error.log         # API errors (if any)
```

**File size target:** Under 100 MB. A 60-second 1080p avatar video from Synthesia typically renders at 50–80 MB before our Remotion wrap. The final CRF 24 encode adds the brand layers; keep total under 100 MB.

## Fallback: when Synthesia is unavailable

| Condition | Fallback |
|-----------|----------|
| Synthesia API down / render stalled > 30 min | Skip avatar segment; build chart-only `data_viz_video` format instead (same data, no talking head) |
| API quota exhausted for the month | Same fallback as above; notify Matt; do not overage-charge without approval |
| Matt's personal avatar rejected / corrupted | Use configured stock avatar + third-person script framing; note the change in `render_log.txt` |
| Script fails voice rules check | Block the render; fix the script; do not proceed with a flagged script |

**Veo 3 is NOT a drop-in fallback for the avatar format.** Veo 3 generates cinematic B-roll, not talking-head presenters. If the avatar pipeline fails, the content type changes — communicate that to Matt before publishing a non-avatar version labeled as a market update.
